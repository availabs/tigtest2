1.0            Introduction

Welcome to the NYMTC Transportation Information Gateway (the Gateway).

 

This is a tool designed to house and visualize various datasets useful for collaborative planning. It contains an array of useful features, which are described in the following document.

 

If you click the "?" icon on any page, you will be directed to a specific place in this document that will give you contextually-appropriate explanations and guidance as to the features on that page.

 

             2.0            User Accounts

                     2.1            Sign Up

Please enter information as completely as possible. Asterisked fields are required.

 

If you've forgotten your password, please click the "Forgot your password?" button, and see Password Reset.

 

                     2.2            Sign In

Enter your username and password to log in to the Gateway. You may click "Remember Me" to be automatically signed in when you visit the Gateway website in the future. If you do not have an account on the Gateway, see Sign Up. If you've forgotten your password, see Password Reset.

 

If you're a public user and do not have an account, you may still be able to see some data sources. However, you will have access to more Sources and Views if you register for an account.

 

                     2.3            Password reset

If you have forgotten your password, click the 'Forgot your password?' button, located in the Sign In screen.

 

Enter the email address you used to sign up for the Gateway, and hit send. The Gateway will send you an email with directions to reset your password.

 

System and Agency Administrators can also aid you in resetting a password.

 

 

             3.0            Dashboard

                     3.1            Overview

Upon logging into the Gateway, you'll see a variety of panes. The Catalog Summary will contain an abbreviated list of the sources and views you have permission to see. Recent Activity will show you a list of recent actions taken by Gateway users. For Administrators, New Contributions will show Sources and Views that have been created or updated recently. My Snapshots will show a list of the sources and views for which you've created a Snapshot.

 

In the header, on the right, there will also be a menu with various options. Clicking your name will display a list of options for customization. If you're an admin, contributor, or librarian, there will be options there for you as well. See My Pages for more information on these options.

 

At any point you can return to the dashboard by clicking the NYMTC Gateway Logo or clicking the "Dashboard" button at the far left of the breadcrumbs menu.

                     3.2            Navigation

The dashboard is an overview of the content in the Gateway. Clicking on the title of any individual pane will show you more information about that subsection.

 

             4.0            Catalog

The Catalog is a collection of all the views and sources in the Gateway. It is found on the Dashboard and can be expanded by clicking either Catalog in its title or clicking the "show more" button at the bottom.

 

Clicking a source will expand all the views available for each source. Clicking on a view will display that view's associated actions in the sidebar next to the view. Different user privileges will see different actions. For the typical user, when available based on data set type, Maps, Tables, Charts, and Metadata will be available.

 

The Catalog can also be listed in Table Format.

 

There are several icons associated with sources and views:

 

If you see a chat bubble icon with a number in it next to a source or view, that means users have commented on that source or view.

 

An eye next to a source or view means you're Watching that item and have additional options.

 

A cloud icon next to a source or view means you're a Contributor and have additional control.

 

A book icon next to a source or view means you're a Librarian and have additional control.

 

             5.0            Recent Activity

Recent activity can be found on the Dashboard.

 

Clicking Show More or Recent Activity will expand the window. It provides a list of recent actions taken by Gateway users.

 

Recent activity provides a timestamped record of the most recent ten events, including: Additions and Updates to Sources and Views and User Comments.

 

The number of entries shown can be customized under My Settings.

 

 

             6.0            New Contributions

New Contributions is a list of datasets that have been recently created or updated by a Contributor, Librarian, or System Administrator. Clicking on any of the links will take you to the Metadata for that Source or View.

                     7.1            Area Selector

The Area Selector is found above the Map. It allows you to drill down a specific area as appropriate to the Source and View. It also allows you to access your defined Study Areas from the Map interface.

 

                     7.2            Other Controls Above Map

Time Selector

The Time Selector is found above the Map in some Sources and Views. It allows you to drill down a specific timeframe as appropriate to the Source and View. For some Sources and Views it will be years, for others it may be time of day, etc.

 

 

Measure Selector

The Measure Selector is found above the Map in some Sources and Views that have multiple measures of data that can be visualized. It is labeled by the "Column:" tag.

 

                     7.3            Sidebar

The sidebar is located on the left side of the Map app. It contains several tools.

Legends

The Legends tab is located to the left of the Map. For currently displayed layers, it elaborates on symbology displayed on the map. To change symbology, see Styles and Symbology.

 

 

Styles & Symbology

The Styles and Symbology tab is located to the left of the Map. It allows users to change current map symbology. Users can adjust colors, numbers of bins, etc. depending on the source.

 

Contributors and Librarians can also save named symbologies for others to use.

 

You can also save your symbologies to use again or share by creating a snapshot. A snapshot is a "save" of the app in its current state. For more information, see Snapshot.

 

Filters

The Filters tab is located to the left of the Map. It allows users to adjust what content is being displayed on the map. Available filters depend on view.

 

Add/Remove Layers

The Add/Remove layers tab is located on the left of the map. It allows users to dynamically add and remove data sets from the map. The icon will change to display the current number of layers active.

Export/Download

The Export/Download tab is located to the left of the map. It allows users to export each layer as a shapefile for use external to the Gateway.

Restore Default Map

The Restore Default Map button is located to the left of the map. It brings back all the default settings of that specific map per dataset.

 

Attribution

The attribution tab is located to the left of the map. It provides information about the map source and contents as defined in the metadata. For more information on metadata, see Metadata.

 

 

                     7.4            Standard Controls

Zoom

The zoom controls allow users to pan in out within the map frame. The plus sign increases zoom, the minus decreases, and the slider in between those two buttons shows the relative zoom level. The slider can be adjusted as well.

 

Full Screen

The full screen button allows the map to assume the entire screen size. Pressing escape will restore the map to normal size.

 

Home Map View

The Home Map View button re-centers the map view to the default zoom and location.

 

Map History

The Map History controls let you go back and forth between zoom and location settings. Their control is similar to how a web browser's back and forward buttons work.

Search Box

On some maps, a search box will filter for keywords based on dataset. It will zoom and relocate the map accordingly.

 

Layer Control

The Layer Control is located at the top right of the map. Available layers can be turned on and off by checking or unchecking the box next to the layer's name.

                     7.5            Study Areas

Study Areas are designed to help you drill down onto specific regional content. To begin defining a study area, click this button on the map.

 


Construct a polygon to define your study area. Click back on the first point to close the shape. There are other controls on the Study Area button to 'Delete last point' and 'cancel.' Study areas lines will include all of any GIS region that their lines intersect with.

Once the study area is defined, a box emerges to name, add comments, or share the study area.

 

You can select defined study areas (either shared to you by someone else or your own creation) by selecting it under the "area" dropdown. See Area Selector.

 

You can also access your study areas by clicking "My Study Areas" after clicking on your name at the top of the page. See My Study Areas.

 

             8.0            Tables

                     8.1            Area Selector

The Area Selector is found above the Table. It allows you to drill down a specific area as appropriate to the Source and View. It also allows you to access your defined Study Areas from the Table interface.

 

                     8.2            Additional Controls Above Table

Some datasets will have additional controls above the table. In this example, there is a button that says "Show on Map." This will take the current filtered table and apply those constraints to the map view. This also can be accomplished using the "Breadcrumbs" to the top left of the page. Clicking the green button where it says Table will also show additional Apps that dataset can be viewed in.

 

There is also a button on this dataset to clear all currently applied filters.

 

                     8.3            Table Controls

Search Box

The search box will filter for alphanumeric strings based on the dataset. It will only display rows containing that string.

 

Range Filters

Range filters allow users to filter the table to contain only rows with a value that falls within the specified range.

Number of Entries to Show

This box allows the user to customize how many rows are displayed at a time in the Table.

 

Column Sorting

Columns can be sorted alphanumerically by clicking the set of antiparallel arrows to the right of each column header. Pressing it once will sort ascending, pressing it a second time will sort descending. Only one column can be sorted at a time, but filters and searches can be applied as well.

Column Filtering

In some datasets, columns can be filtered. There will be a dropdown box in the column header where you can select a specific entry to be filtered upon. In this case, each column can have a specific filter (as opposed to sorting, where only one column at a time can be sorted).

                     8.4            More

Clicking the More button reveals a set of three new options.

Export Filtered

This button lets you download a .csv copy of the table, keeping current filters active.

Export All

This button lets you download a .csv copy of the table, ignoring filters. You will download the entire data set. Note that SpeedFacts datasets, like NPMRDS and TRANSCOM, cannot be exported in their entirety due to size.

Show / hide columns

If you prefer not to see certain columns at all, you can disable them in the show/hide columns dialog. Columns can be restored by returning to this menu.

    10.0            Snapshots

Registered Users can save a snapshot of the current state of a map, table, or chart so that it can be recalled later without having to reapply filters or other customizable features.

 

Snapshots are stored for you and can be shared with others during creation.

 

To create a snapshot, click the "Take Snapshot" button at the top right of a map, table, or chart you've customized. If you have a snapshot (created by you or shared with you) of the specific View and App already, an asterisk will appear next to the "Take Snapshot" with the number of snapshots currently associated with that View and App.

 

You can view a list of all your snapshots in My Snapshots or on the Dashboard.

 

    11.0            Comments

Registered users can contribute comments on Sources, Views, and in individual Apps. Users can add comments wherever there is a button near the top right navigation menu that says "Add Comment."

 

Comments you make will be visible to all users unless marked Admin Only.

 

In the Catalog, you'll see little chat bubbled with numbers in them. This indicates that a user has left a comment on that particular Source, View, or App. You can hover over each icon to see a summary of comments. Clicking the icon will show you all of the comments in full.

               11.1            Adding Comments

To add a new comment to a Source in the Catalog, click the arrow to the right of the title. A menu will appear. Click "Add Comment."

 

To add a new comment to a View in the Catalog, click the View, then click "Add Comment" in the Actions pane that appears.

 

To add a new comment inside an App, click the "Add Comment" button to the top right of the app, below the top right navigation menu.

 

In the New Comment window, add a subject (mandatory) and the body of your text. It will appear next to whichever Source, View, or App (shown at the top of the Add Comment window) with a timestamp. If you are an administrator, you have the option to make your comment visible only to other administrators.

Metadata is a collection of information that helps describe a particular Source or View. Metadata can tell you in-depth information about the topic, like who contributed the data, when it was created, when it was last edited, the format of the data, etc.

 

It can be accessed from the Catalog. Click a particular Source or View, then select View Metadata from the Apps panel.

    16.0            Access Controls

Access Controls allow you to more directly manage what is allowed to be seen/downloaded/commented on, and by whom, in the Gateway application.

 

Users are grouped in a few ways. In the 'General' section you will find:

-          Guest User someone who visits the application, but does not have an account and is not logged in.

-          Public User someone who has an account and is logged in, but is a base-level user (i.e. is not associated with an Agency/has no special privileges)

-          Agency User someone who has an account, and is assigned to an Agency (as an Admin, User, Contributor, etc.)

 

By default, the 'Agencies/Users' section will be blank. If you wish to have more granular control over which agencies can see a certain Source or View, click the Add Agencies link on the right-hand side of the page. An entry will be added to the form, and you can modify its settings there.

 

If you wish to modify the access for a specific agency's user, once you have added the user's agency to the form, you can click on Add Users, then their Agency, and then select the desired user.

 

 

In a view, you will also have an option at the bottom near "Save" to "Use Source Access Controls" this will make access for that View exactly the same as the Source.

 

The Reset button allows you to restore default Access Controls.

 

 

 

IMPORTANT NOTES

-          Full System Admins cannot be restricted, and thus have constant access to all Sources and Views in the system.

-          If a user has multiple roles, the system will give them the abilities of their [most powerful/least restrictive] role. The hierarchy, from most to least powerful, is as follows:

Admin > Agency Admin > Agency User > Contributor > Public

e.g. If a user has the Public, Agency User, and Agency Admin roles, they will be given Agency Admin access.

 

In almost all cases, more specific settings will override more general settings. HOWEVER, if a Source is not visible/downloadable/commentable, its Views will also not be. You can also prevent a View from being accessed by changing its Access Controls specifically.

               17.9            Update Help

System administrators can modify the content in this window by uploading a new Gateway Help document. Currently, functionality for replacing images is not supported. The steps to replace text are as follows:

1.       Log in to Gateway as Admin.

2.       From either All Uploads or "Update Help" download most recent help GatewayHelp.docx file.

3.       Edit in MS "Word".

4.       Save modified .docx file.

5.       File -> Save As -> Browse then in Save As dialog for "Save as type" select "Web Page, Filtered (*.htm, *.html)".

6.       In Gateway, select "Update Help" and upload modified .docx file.

7.       Select "Update Help" again and upload modified .htm file.

IMPORTANT NOTES:

1.       Make sure that any 'smart' quotation marks that Word adds are replaced with the standard version use ' and " in a find/replace to make sure none are included.

2.       To allow the website to hyperlink to the correct section of the help document, hover over the "?" on the Gateway page and note the text after the # in the hyperlink. In the Word help document, highlight the section header that will correspond to that page and go to Insert -> Bookmark. Find the text after the # from the hyperlink and select that bookmark. Add a new one if necessary.

3.       Changing images is currently not supported without additional assistance.

 

 

    18.0            Uploading Data

To upload, create and process datasets, you must have contributor and librarian privileges, which can be granted by the owner of the respective datasets or the Gateway Admin.

18.1 SED COUNTY AND TAZ

For SED county and TAZ datasets, you will have to create new SOURCES for each new year dataset. The steps to create new datasets are as follows:

 

1.      Click on the Main Catalog window or click expand all at the bottom of the main home page.

2.      Click on New Source as shown by the red line below.

3.      Fill in all the details of the new dataset and click save. It will take you to the metadata page.

4.      Now you can upload the new dataset to this source. Click on the tab "Contributor" on the top right next to the log in buttons. You will see all the datasets that you have contribution privileges.

5.      Click on the Upload option in the dropdown menu of the source that you just created as shown below.

 

 

6.      Click Upload and in the upload page add the SED csv file. The boundary file is already selected in the Gateway. If there has been an update in the boundary file, you can select from the Area Boundary Base Year option shown below.

7.      Once the upload completes, it will take you to the metadata page from where you can process the file.

8.      From the metadata page, click "Uploads" hyperlink from the Updates section as shown below.

 

9.      In the Uploads page shown below, click Process on the correct file and the new dataset will become available once the processing completes.

 

NOTE: For SED data, this process creates all the new views automatically once the processing completes.

18.2 ACS/CENSUS DATA

For ACS datasets, you will first have to download the files from the ACS Census website and then upload to the Gateway.

 

Each year there are new tables that become available in the ACS website. The two files that are available through the Gateway website are:

1. Absolute and Relative Minority Population Data

2. Absolute and Relative Population Below Poverty Level

 

18.2.1 DOWNLOADING FROM ACS WEBSITE

 

 

Absolute and Relative Minority Population Data (ACS Table id: DP05)

18.2.1 UPLOADING AND PROCESSING IN THE GATEWAY

To upload new datasets, you will have to create new views for the ACS datasets.

1.      Click on a view for which you want to create a new data view.

2.      Click Copy from the Action items on the right

3.      Fill all the mandatory spaces in the new view as shown below. Give the view a new name similar to the one you copied. Click save and it will take you to the metadata page.

4.      You can now upload the files either through the metadata page or the main catalog page.

5.      In the Catalog page, click on the new view you just created and click upload from the Action items.

 

6.      From the metadata page, click upload from the dropdown option on the top menu bar under "View Metadata" as shown below.

7.      Click Upload and upload the appropriate csv file, enter the required information in the upload page and click save.

8.      The new upload can now be viewed from the "Updates" section in the metadata page.

9.      Click on the "Uploads" link under the Updates section and it will take you to a page with all the uploads made to this new view.

10.  Click Process on the correct file and the new dataset will become available once the processing completes.

11.  Repeat this process for the two ACS files downloaded from the ACS website.

18.4 HUB-BOUND/NPMRDS/TRANSCOM DATA SETS

 

The Hub Bound dataset is an annual dataset and can be uploaded only once a year. NPMRDS and TRANSCOM datasets are monthly datasets which can be processed and uploaded every three months. The steps to upload these datasets into Gateway are outlined below:

 

1.      Click on the respective view for which you want to upload a new dataset.

2.      Click on Upload from the Action items on the right.

3.      Upload the csv/zip file (for NPMRDS and TRANSCOM) and access database (for Hub Bound), enter the year/month and Notes for the dataset as shown below and click save.

 

 

4.      Once uploaded, it will take you to the metadata page where you can go to the Updates section and click on "Uploads" hyperlink.

5.      In the Uploads page, click "Process" on the most recent file that you just uploaded. You can also sort the datasets by the "Updated at" field in the table to get the most recent uploaded file.

6.      You can view the progress bar by clicking on the "Process"/ "Processing" hyperlink in the upload page and it will take you to another page where you can view the processing progress bar.

 

 

    19.0            Frequently Asked Questions

               19.1            Why can't I see sources or views on the dashboard?

Are you logged in? Guest users (users who are not logged in) have limited access to Sources and Views. Either log in or sign up for an account.

 

If you are already logged in, but can't see any Sources or Views, or can't find one that you believe you have access to, contact your agency administrator. Agency administrators have control over which users and agencies can see Sources and Views that they contribute to the Gateway.

               19.2            Why am I having trouble seeing Gateway content?

The Gateway only supports modern web browsers. Please contact your IT Staff to determine if your browser is recent enough to view HTML5 content. The Gateway will not function correctly with IE9 and prior.

 

               19.3            Is there data for <Insert Region> in the gateway?

Agency Administrators provide data to those that they find qualified to view it. If there's a specific region in the NYMTC area whose data you'd like to see, please contact their agency administrator.

               19.4            When I switch between Sources, Views, and Apps, my filters persists. Why?

Where applicable, filters carry through to different applications, sources, and views. We expect users to generally want to be looking at similar data subsets within the same general time span. Remembering filter settings is generally considered more useful than requiring the user to set the filters up again every time they navigate to a different view.

               19.5            How do Study Area borders intersect with GIS borders?

Study area lines that intersect with GIS borders are inclusive they will include all of the data for any delineation they intersect.

               19.6            Why do NPMRDS and TRANSCOM data take a long time to process?

These data sets in particular are very large. Mapping, tabularizing, or charting them takes a considerable amount of resources. Exporting, uploading, and processing them will also take a considerable amount of time.
